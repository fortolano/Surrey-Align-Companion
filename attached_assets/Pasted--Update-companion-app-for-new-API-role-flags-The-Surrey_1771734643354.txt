  Role hierarchy (who sees what)

  These flags are not mutually exclusive — a user can have multiple true flags. Here's how they nest for access control:

  - Full access: is_stake_admin
  - Stake-wide visibility: is_stake_president, is_stake_presidency_member, is_high_councilor
  - Stake council participants: is_stake_council_member (includes presidency, HC, stake org presidents, directors)
  - Ward-wide visibility: is_bishop, is_bishopric_member
  - Organization-level: is_ward_org_president, is_ward_org_presidency_member, is_stake_org_president, is_stake_org_presidency_member
  - Support roles (same tool access as their presidency): is_executive_secretary, is_stake_director

  What to do

  1. Search and replace any reference to is_stake_presidency with is_stake_presidency_member
  2. Update the user type/interface to include all new boolean fields
  3. Update any role-based UI logic (showing/hiding features, navigation items, etc.) to use the new granular flags where appropriate
  4. Executive secretaries (is_executive_secretary) should generally have the same tool access as their presidency members — they handle scheduling, agendas, and coordination
