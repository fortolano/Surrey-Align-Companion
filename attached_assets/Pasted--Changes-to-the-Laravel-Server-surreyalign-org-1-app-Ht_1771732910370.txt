
  ---
  Changes to the Laravel Server (surreyalign.org)

  1. app/Http/Controllers/ApiExternal/ReferenceDataController.php

  What changed: The currentHolders() method now accepts an optional ?ward_id=N query parameter. When provided, it filters holders to only that ward. It
   also returns ward_id in each holder's response payload. Previously it returned holders from ALL wards in the stake with no filtering.

  2. app/Http/Controllers/ApiExternal/AuthController.php

  What changed: The formatUser() method now returns three additional role flags: is_high_councilor, is_bishop, is_bishopric_member. These use existing
  methods on the User model. Previously only is_stake_admin and is_stake_presidency were returned.

  ---
  Changes to the Replit Mobile App

  Core Libraries

  3. lib/api.ts — Complete rewrite

  What changed: Replaced the simple authFetch function with a version that:
  - Detects 401 responses (expired tokens) and clears stored credentials
  - Calls a global onAuthExpired handler so the auth context can reset state
  - Exports setAuthExpiredHandler() for the auth provider to register its callback
  - Imports SecureStore and AsyncStorage to clear credentials on 401

  4. lib/auth-context.tsx

  What changed:
  - Added is_high_councilor, is_bishop, is_bishopric_member to the SAUser interface
  - Added import { setAuthExpiredHandler } from '@/lib/api' and a useEffect in AuthProvider that registers a handler to clear user/token state when a
  401 is detected
  - Added export function useLogout() hook that clears auth state, clears the React Query cache, then redirects — using window.location.href = '/' on
  web (hard page reload) and router.replace('/') on native
  - Added import { router } from 'expo-router' and import { queryClient } from '@/lib/query-client'

  5. lib/query-client.ts

  What changed: Removed the getQueryFn function, apiRequest function, throwIfResNotOk helper, the fetch import from expo/fetch, and the QueryFunction
  type import. Also removed queryFn: getQueryFn({ on401: "throw" }) from the default query client options. These were all unused — every screen uses
  authFetch directly.

  New Constants Files

  6. constants/layout.ts — NEW FILE

  What it does: Exports WEB_TOP_INSET (67 on web, 0 on native) and WEB_BOTTOM_INSET (34 on web, 0 on native). Replaces hardcoded Platform.OS === 'web'
  ? 67 : 0 definitions that were duplicated across 7+ screens.

  7. constants/status-colors.ts — NEW FILE

  What it does: Exports the STATUS_COLORS constant (a map of calling request status → background/text color pairs). This was previously defined
  identically in both callings.tsx and calling-detail.tsx.

  New Components

  8. components/ScreenHeader.tsx — NEW FILE

  What it does: A shared teal header component with title, optional subtitle, and optional right action button. Provides consistent header styling
  across screens.

  9. components/SkeletonCard.tsx — NEW FILE

  What it does: Exports SkeletonCard and SkeletonList components with animated shimmer. Ready to replace the <ActivityIndicator> loading states on list
   screens.
